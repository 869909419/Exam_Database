# 考公资料库项目逻辑

## 目标

本项目建设一个以 Obsidian 为主界面的考公资料库。首版 MVP 优先跑通“公开来源采集 -> 正文清洗 -> Markdown 入库 -> SQLite 索引 -> Obsidian 检索/复习视图”，并为真题导入、题型分类、刷题记录预留稳定接口。

资料库服务于三类核心工作：

1. 政策理论和时政材料积累：从求是网、人民网评、中国政府网、新华网时政、人民日报/人民网、统计局、地方政府网站等公开来源采集文章，清洗成可长期阅读和引用的 Markdown。
2. 真题题库建设：导入公开可下载或用户自备真题 PDF/Markdown，逐步转换成结构化题目卡片。
3. 刷题复盘：记录作答、错因、耗时、题型分布，并生成 Obsidian 可查询的错题和周报。

## 模块边界

- Obsidian vault 是用户阅读、检索、复习的主入口。
- Python CLI 负责确定性流程：采集、清洗、去重、索引、导入、统计。
- DeepSeek/Claude Code 负责非确定性辅助：标签建议、正文质量检查、题型分类、解析生成、错因总结。
- SQLite 是结构化索引和统计数据库，不替代 Markdown 的长期可读性。

## 数据流

1. Source adapter 从公开网页或用户文件读取原始材料。
2. 原始 HTML/PDF/文本保存到 `data/raw/`，用于审计和重复处理。
3. 清洗器移除导航、广告、推荐阅读、版权尾巴等噪声。
4. 正文图片下载到 vault 内部归档目录，Markdown 使用本地相对路径引用。
5. 结构化记录写入 SQLite，同时输出带 YAML frontmatter 的 Markdown 到 `vault/`。
6. Obsidian 通过普通目录、标签和 Dataview 查询展示资料、题目和刷题统计。

## 合规边界

- 只处理公开可访问内容、公开下载文件和用户自备文件。
- 不绕过登录、验证码、付费墙、反爬限制或访问控制。
- 商业题库网站只在公开页面或用户明确提供材料时处理。
- 每条采集记录必须保留来源 URL、采集时间、内容 hash 和原始文件路径。

## 第一阶段完成标准

- 仓库包含可打开的 Obsidian vault 目录。
- `examdb` CLI 能初始化数据库、采集/导入样例内容、写出 Markdown。
- 求是网 article adapter 有可测试的解析和清洗逻辑。
- 求是网文章按期数归档，图片保存在同一期文件夹下的 `附件/`。
- 四个项目内 skill 能指导 agent 执行采集、导入、分类、复盘。
- 单元测试覆盖清洗、Markdown frontmatter、去重入库、题库导入骨架。
