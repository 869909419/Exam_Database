# 来源 Adapter 备忘录

## 下一批候选

| Source ID | 来源 | 价值 | 初始范围 |
| --- | --- | --- | --- |
| `stats-gov` | 国家统计局 | 统计公报、经济数据、图表材料 | 统计公报、数据解读，不抓复杂下载表格 |
| `people-daily` | 人民日报公开页面 | 重要评论、理论版、规范表述 | 先做评论/理论/重要版面，不做全站 |
| `mohrss` | 人力资源和社会保障部 | 就业、社保、人才、劳动权益 | 政策文件、政策解读、新闻发布 |
| `ndrc` | 国家发展改革委 | 宏观经济、产业政策、区域协调、新质生产力 | 政策文件、解读、新闻发布 |
| `mee` | 生态环境部 | 生态文明、美丽中国、绿色转型、污染防治 | 政策文件、解读、例行发布 |
| `mca` | 民政部 | 基层治理、养老服务、社会救助、社区建设 | 政策文件、解读、工作动态 |
| `gmw-theory` | 光明网理论 | 理论文章、文化治理、思想材料 | 理论频道公开文章 |

## 建议顺序

1. `stats-gov`：先补数据材料，和已有政策/评论来源形成互补。
2. `people-daily`：补人民日报公开评论和理论类规范表述。
3. `mohrss`、`ndrc`：就业民生和宏观经济是申论高频主题。
4. `mee`、`mca`：生态文明、基层治理、养老和社会救助可作为专题材料库。
5. `gmw-theory`：作为理论文章补充来源。

## 落地原则

- 每个来源先做 2-3 个稳定公开栏目，不做全站泛抓。
- 优先找官方列表页、JSON 数据源或 RSS；没有稳定列表时才用首页链接发现。
- 商业站、登录页、验证码、付费墙不纳入自动采集。
- 新 adapter 必须补 fixture、真实 smoke test、Obsidian wrapper 和文档规则。
